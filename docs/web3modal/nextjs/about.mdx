# Next.js

Web3Modal SDK has support for [Wagmi](https://wagmi.sh/), [Ethers v5](https://docs.ethers.org/v5/getting-started/) and
[Ethers v6](https://docs.ethers.org/v6/).
Choose one of these ethereum libraries to get started.

:::info Note
These steps are specific to [Next.js](https://nextjs.org/) app router.
:::

## Extra configuration

Next.js relies on [SSR](https://nextjs.org/docs/pages/building-your-application/rendering/server-side-rendering). This means some specific steps are required to make Web3Modal work properly.

- Add the following code in the `next.config.js` file

```ts
// Path: next.config.js
const nextConfig = {
  webpack: config => {
    config.externals.push('pino-pretty', 'lokijs', 'encoding')
    return config
  }
}
```

- If `SWCMinify` is flagged, it must be false in the `next.config.js` file
- If you are using Wagmi, [dynamic imports](https://nextjs.org/docs/app/building-your-application/optimizing/lazy-loading#nextdynamic) or `supressHydrationWarning` are recommended tro prevent `Hydration mismatch` errors
